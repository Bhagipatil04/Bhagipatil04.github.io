Deployment of Java Application using AWS (VPC Three-Tier)

01

Designed and deployed a Java application on a secure three-tier VPC architecture, separating web, application and database layers across subnets.

AWS VPCEC2RDSJava
Problem Statement

The application needed a production-style deployment with isolated network tiers instead of a flat, single-subnet setup that exposes the database directly to the internet.

Architecture Overview

A custom VPC with public subnets for load-facing EC2 instances, private subnets for application servers, and isolated subnets for the RDS database — connected through NAT gateways and route tables.

Technologies Used

AWS VPC, EC2, RDS, Security Groups, NAT Gateway, Java, Apache Tomcat.

Key Implementations

Configured subnet-level network ACLs, security groups scoped by tier, and a bastion host for secure access to private instances.

Outcomes / Learnings

Strengthened understanding of network segmentation and least-privilege access design in real AWS environments.
